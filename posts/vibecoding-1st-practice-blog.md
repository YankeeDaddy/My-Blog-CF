# 文科生的Coding梦

作为文科生，一直有一个强烈的愿望—————能亲手开发出自己的一个软件产品。为此七年前专门系统学过python，由于能力和精力有限，最后也没能给自己一个交代。如今AI时代，在WorkBuddy的帮助下，我前后花了三天时间就完成了。PeyBlog就是我人生的第一个Coding作品。

# VibeCoding感受

VibeCoding，让我的coding梦想照进了现实。说几点我的感受吧。

- 需要一定的IT基础知识。我之所以能这么快地完成，跟我之前对html/JS/GitHub/网络/域名等知识有过一定的了解，有很大的关系。否则光是GitHub的注册和访问都会是一个大问题，毕竟国内访问GitHub很慢。
- 个人判断力很重要。在VibeCoding的过程中，会有各种问题出现，比如环境、工具等，这是Agent就会自己绕过问题去找其他解决办法。这个过程中，很可能会走弯路。技术栈选择、产品架构等方面，更是如此。
- Agent的测试和验证能力较弱。个人博客这个网站，第一个版本其只花了一个小时就完成了。麻烦的是后面的功能优化和Bug修复。印象最深的是如何实现文章点赞数据的持久化问题。至少尝试了三种方案，修复Bug尝试至少二十次。我的耐心都快被磨没了，好在最终还是验证成功了。这也验证了个人判断力的重要性。在它给出方案、修改代码文件的时候，由于不懂，我基本没有任何判断力。
- 技术选型超级重要。一开始用的是Netlify托管网页服务，可是由于不断修改代码，反复构建，第二天免费额度就用完了。后来对比研究，才切换到CloudFlare。再后来CloudFlare的免费域名在国内被污染得很严重，导致没法稳定访问，花了10美元购买了peyblog.com域名。如果一开始充分调研技术选型的话，就不会走Netlify这个弯路了。当然换个角度来说，对我这样的新手，多了解一种方案、一个平台，是件好事。

## 附：系统架构

PeyBlog 采用「**纯静态 SPA + 无服务器云服务**」架构，由三个核心组件构成：

```
┌──────────────────────────────────────────────────────────────┐
│                        用 户 浏 览 器                          │
│  ┌─────────────────────────────────────────────────────────┐ │
│  │              index.html (SPA, ~2000 行)                   │ │
│  │  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌───────────┐  │ │
│  │  │ 文章浏览  │ │ Markdown │ │  点赞系统 │ │  Giscus   │  │ │
│  │  │ (读者侧)  │ │  编辑器   │ │ (持久化)  │ │  评论组件  │  │ │
│  │  └──────────┘ └──────────┘ └──────────┘ └───────────┘  │ │
│  └─────────────────────────────────────────────────────────┘ │
└──────────────┬──────────────────────┬────────────────────────┘
               │                      │
     ┌─────────▼─────────┐  ┌────────▼──────────┐
     │ Cloudflare Pages  │  │ Cloudflare Worker  │
     │ (静态托管 + CDN)   │  │  peyblog-likes-api │
     │                   │  │  api.peyblog.com   │
     │ • index.html      │  │                    │
     │ • posts/*.md      │  │ GET  /api/likes    │
     │ • posts/index.json│  │ POST /api/likes    │
     │ • posts/likes.json│  │ GET  /api/discuss..│
     │ • _headers        │  │                    │
     └────────┬──────────┘  └────────┬──────────┘
              │                      │
              │        ┌─────────────┴─────────────┐
              │        │       GitHub API           │
              └────────┤  ┌──────────┐ ┌─────────┐ │
                       │  │ REST API │ │  GraphQL │ │
                       │  └────┬─────┘ └────┬────┘ │
                       └───────┼────────────┼──────┘
                               │            │
                    ┌──────────▼──┐  ┌──────▼──────────┐
                    │ My-Blog-CF  │  │My-Blog-Comments-CF│
                    │  (主仓库)    │  │   (评论仓库)      │
                    │             │  │                  │
                    │ posts/*.md  │  │ Discussions      │
                    │ likes.json  │  │ (Giscus 使用)    │
                    │ index.json  │  │                  │
                    └─────────────┘  └──────────────────┘
```

### 组件职责

| 组件                      | 职责                                      | 关键技术                             |
| :------------------------ | :---------------------------------------- | :----------------------------------- |
| **SPA 前端** (index.html) | 路由、渲染、编辑器、点赞交互、Giscus 集成 | Vanilla JS + Marked.js               |
| **Cloudflare Pages**      | 静态文件托管、全球 CDN 分发、自动部署     | Cloudflare CDN                       |
| **Cloudflare Worker**     | 点赞数据读写 API、评论数聚合 API          | Cloudflare Workers + GitHub REST API |
| **GitHub 主仓库**         | 文章源文件存储、版本控制、点赞数据持久化  | Git + GitHub REST API                |
| **GitHub 评论仓库**       | Giscus 评论数据（Discussions）            | GitHub Discussions + Giscus App      |
