## 一、前端模块详解

### 1.1 文件结构

```
index.html  (~2028 行，单文件 SPA)
├── <head>              # 元数据、Marked.js CDN 引入
├── <style>             # 完整 CSS（~330 行），CSS 变量主题系统
├── <body> HTML         # 页面骨架（导航、路由容器、编辑器骨架）
└── <script>            # 完整 JavaScript（~1600 行）
    ├── 常量 & 状态
    ├── 路由系统        # Hash Router（#/home, #/post/{slug}, #/editor, #/about）
    ├── 文章数据层      # loadArticlesMeta, loadArticleBody, Frontmatter 解析
    ├── GitHub API 层   # githubAPI, githubPushFile, pushArticle, deleteArticle
    ├── 点赞系统         # loadLikesRemote, syncLikesToGithub, handleLike
    ├── 评论系统         # Giscus 加载、主题同步
    ├── 渲染层           # renderHome, renderDetail, renderEditor, renderAbout, renderSidebar
    ├── 编辑器           # buildEditorUI, saveArticle, insertMD, onEditorInput
    ├── 主题系统         # initTheme, toggleTheme, syncGiscusTheme
    └── 工具函数         # escapeHtml, showToast, showModal
```

### 1.2 路由设计（Hash Router）

```
URL Hash                 → 视图              → 渲染函数
────────────────────────────────────────────────────────
#/home                   → 首页文章列表        → renderHome()
#/post/{slug}            → 文章详情页          → renderDetail(slug)
#/editor                 → Markdown 编辑器     → renderEditor()
#/editor/{slug}          → 编辑已有文章        → renderEditor(slug)
#/about                  → 关于页面            → renderAbout()
```

**管理模式入口**：`?edit=1` URL 参数，激活 `ADMIN_MODE`，显示编辑器入口和 GitHub 配置。

### 1.3 数据流

```
                        用户操作
                           │
          ┌────────────────┼────────────────┐
          ▼                ▼                 ▼
    浏览/搜索/筛选      点赞/取消点赞      写文章/发布
          │                │                 │
          ▼                ▼                 ▼
    posts/index.json   localStorage       Markdown 编辑器
    (文章元数据索引)     (liked 状态)       (Frontmatter + 正文)
          │                │                 │
          ▼                ▼                 ▼
    GET /api/likes     本地状态更新        GitHub Contents API
    GET /api/discuss..  (UI 即时反馈)      PUT posts/{slug}.md
          │                │              PUT posts/index.json
          ▼                ▼                 │
    合并点赞数和评论数   POST /api/likes      ▼
          │           (后台同步到 GitHub)  Cloudflare Pages
          ▼                │              自动部署（~30s）
    渲染文章卡片           ▼
                       Worker → GitHub
                       PUT likes.json
```

### 1.4 点赞系统详细设计

#### 数据存储层次

```
┌─────────────────────────────────────────┐
│            点赞数据存储                    │
│                                          │
│  Layer 1: localStorage (浏览器本地)      │
│  ┌────────────────────────────────────┐  │
│  │ key: "peyblog_likes"              │  │
│  │ value: { "slug": count, ... }     │  │
│  │ 作用：快速读取、离线可用、           │  │
│  │       记录 liked 状态               │  │
│  └────────────────────────────────────┘  │
│              ↕ syncLikesToGithub()        │
│  Layer 2: GitHub (服务端持久化)          │
│  ┌────────────────────────────────────┐  │
│  │ posts/likes.json                   │  │
│  │ value: { "slug": count, ... }      │  │
│  │ 作用：跨设备数据源、Git 版本控制    │  │
│  └────────────────────────────────────┘  │
└─────────────────────────────────────────┘
```

#### 读取优先级

```
loadLikes()
  1. 尝试 Worker API: GET /api/likes
  2. 成功 → 合并 localStorage liked 状态 → 返回
  3. 失败 → 从 localStorage 读取 → 返回
```

#### 写入流程

```
用户点击 ♡
  ↓
incrementLike(slug)           # 更新 localStorage
  ↓
handleLike(slug)              # 更新 UI（即时反馈）
  ↓
syncLikesToGithub()           # 异步后台同步
  ↓
POST /api/likes               # 发送完整数据 { likes: {...} }
  ↓
Worker handlePostLikes()
  ├── GET GitHub REST API     # 读取当前 likes.json 和 sha
  ├── 合并数据（前端数据覆盖）
  ├── PUT GitHub REST API     # 写入 likes.json（含 sha，防冲突）
  └── 返回 { success: true }
```

## 二、Cloudflare Worker 详细设计

### 2.1 Worker 文件结构

```
worker.js (~147 行，ES Modules)
├── export default { fetch() }     # 入口，路由分发 + CORS
├── handleGetLikes(env, headers)   # GET /api/likes
├── handlePostLikes(req, env, hdr) # POST /api/likes
└── handleGetDiscussions(env, hdr) # GET /api/discussions
```

### 2.2 路由表

```
请求                          → 处理函数
────────────────────────────────────────────
OPTIONS (任意路径)             → 直接返回 CORS 头
GET  /api/likes               → handleGetLikes()
POST /api/likes               → handlePostLikes()
GET  /api/discussions         → handleGetDiscussions()
其他                           → 404 Not Found
```

### 2.3 handleGetLikes — 获取点赞数据

```
时序：
  GET /api/likes
    │
    ▼
  Worker.fetch()
    │ (CORS 处理)
    ▼
  handleGetLikes()
    │
    ▼
  GET https://api.github.com/repos/YankeeDaddy/My-Blog-CF/contents/posts/likes.json
  Headers: Accept: vnd.github.v3+json, User-Agent: peyblog-worker
  ⚠️ 无 Authorization（公开仓库，不需要认证）
    │
    ├── 200 → 读取 apiData.content（base64）
    │          atob 解码 → JSON.parse → 返回 JSON
    │          Response headers: Cache-Control: no-store
    │
    ├── 404 → 返回 {}（文件不存在，视为无数据）
    │
    └── 其他 → 500 { error, detail }
```

**设计决策**：使用 `api.github.com` REST API 而非 `raw.githubusercontent.com`，避免 CDN 缓存延迟（Bug #7 修复）。

### 2.4 handlePostLikes — 更新点赞数据

```
时序：
  POST /api/likes
  Body: { likes: { "slug": count, ... } }
      或 { articleId: "slug", action: "like"|"unlike" }
    │
    ▼
  handlePostLikes()
    │
    ├── 1. GET 当前 likes.json (读 sha 和现有数据)
    │      ⚠️ 无 Authorization（公开仓库）
    │
    ├── 2. 合并数据
    │      if body.likes → 完整覆盖
    │      if body.articleId + action → 增量更新
    │
    ├── 3. 构建 PUT 请求
    │      message: "Update likes data"
    │      content: base64(JSON.stringify(mergedData))
    │      sha: (从步骤 1 获取，防并发冲突)
    │      branch: "main"
    │      Headers: Authorization: Bearer ${GITHUB_PAT}
    │               User-Agent: peyblog-worker
    │
    ├── 4. PUT 写入 GitHub
    │      ✓ 200 → 返回 { success: true, likes: mergedData }
    │      ✗ 4xx/5xx → 返回 500 { error, detail }
```

### 2.5 handleGetDiscussions — 获取评论数

```
时序：
  GET /api/discussions
    │
    ▼
  handleGetDiscussions()
    │
    ▼
  GET https://api.github.com/repos/YankeeDaddy/My-Blog-Comments-CF/discussions?per_page=100
  Headers: Authorization: Bearer ${GITHUB_PAT}
           Accept: vnd.github.v3+json
           User-Agent: peyblog-worker
    │
    ▼
  遍历 Discussions 数组：
    • 提取 title → 去除 "#/post/" 前缀 → 得到 slug
    • 读取 d.comments 字段（⚠️ 不是 comments_count！）
    • 构建 { "slug": commentCount, ... }
    │
    ▼
  返回 JSON { "slug": count, ... }
```

### 2.6 GITHUB_PAT 环境变量要求

| 属性 | 要求 |
|------|------|
| **类型** | GitHub Fine-grained Personal Access Token |
| **有效期** | 建议 No expiration |
| **仓库权限** | `YankeeDaddy/My-Blog-CF` → Contents: Read and Write |
| | `YankeeDaddy/My-Blog-Comments-CF` → Discussions: Read |
| **认证头格式** | `Authorization: Bearer github_pat_xxxx`（⚠️ 不是 `token`！） |
| **存储位置** | Cloudflare Dashboard → Workers → peyblog-likes-api → Settings → Variables → GITHUB_PAT（Encrypt） |

```typescript
interface ArticleMeta {
  slug: string;        // URL 友好标识符（基于标题拼音）

  date: string;        // ISO 日期 YYYY-MM-DD
  category: string;    // 分类（如"随想"、"技术"、"设计"）

  excerpt: string;     // 摘要（从 Frontmatter 读取）
}
```

### 4.2 点赞数据（likes.json）

```typescript
// 扁平格式：key 为 slug，value 为点赞计数
type LikesData = Record<string, number>;

// 示例
{
  "victim-thinking": 1,
  "info-and-knowledge": 2
}
```

### 4.3 评论数缓存

```typescript
// Worker /api/discussions 返回值
type CommentCounts = Record<string, number>;

// 示例

  "victim-thinking": 3,
  "info-and-knowledge": 1

```

### 4.4 本地存储结构

| Key | 类型 | 内容 |
|-----|------|------|

## 七、错误处理与容错

| 场景 | 策略 |
|------|------|
| Worker API 不可用 | 回退到 localStorage 缓存的点赞数据 |
| GitHub API 限流 | 前端不阻塞 UI，后台静默重试 |
| 文章数据加载失败 | 显示错误提示，保留导航可用 |
| Giscus 加载失败 | 评论区域显示「评论加载失败」 |
| likes.json 不存在 | Worker 返回 `{}`，前端正常处理 |
| PUT 冲突（并发写入） | Worker 使用 sha 参数保证原子性 |
| 网络断开 | SPA 正常工作（离线缓存），同步时静默失败 |
