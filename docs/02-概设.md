## 一、系统架构总览

PeyBlog 采用「**纯静态 SPA + 无服务器云服务**」架构，由三个核心组件构成：

```
┌──────────────────────────────────────────────────────────────┐
│                        用 户 浏 览 器                          │
│  ┌─────────────────────────────────────────────────────────┐ │
│  │              index.html (SPA, ~2000 行)                   │ │
│  │  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌───────────┐  │ │
│  │  │ 文章浏览  │ │ Markdown │ │  点赞系统 │ │  Giscus   │  │ │
│  │  │ (读者侧)  │ │  编辑器   │ │ (持久化)  │ │  评论组件  │  │ │
│  │  └──────────┘ └──────────┘ └──────────┘ └───────────┘  │ │
│  └─────────────────────────────────────────────────────────┘ │
└──────────────┬──────────────────────┬────────────────────────┘
               │                      │
     ┌─────────▼─────────┐  ┌────────▼──────────┐
     │ Cloudflare Pages  │  │ Cloudflare Worker  │
     │ (静态托管 + CDN)   │  │  peyblog-likes-api │
     │                   │  │  api.peyblog.com   │
     │ • index.html      │  │                    │
     │ • posts/*.md      │  │ GET  /api/likes    │
     │ • posts/index.json│  │ POST /api/likes    │
     │ • posts/likes.json│  │ GET  /api/discuss..│
     │ • _headers        │  │                    │
     └────────┬──────────┘  └────────┬──────────┘
              │                      │
              │        ┌─────────────┴─────────────┐
              │        │       GitHub API           │
              └────────┤  ┌──────────┐ ┌─────────┐ │
                       │  │ REST API │ │  GraphQL │ │
                       │  └────┬─────┘ └────┬────┘ │
                       └───────┼────────────┼──────┘
                               │            │
                    ┌──────────▼──┐  ┌──────▼──────────┐
                    │ My-Blog-CF  │  │My-Blog-Comments-CF│
                    │  (主仓库)    │  │   (评论仓库)      │
                    │             │  │                  │
                    │ posts/*.md  │  │ Discussions      │
                    │ likes.json  │  │ (Giscus 使用)    │
                    │ index.json  │  │                  │
                    └─────────────┘  └──────────────────┘
```

### 组件职责

| 组件 | 职责 | 关键技术 |
|------|------|---------|
| **SPA 前端** (index.html) | 路由、渲染、编辑器、点赞交互、Giscus 集成 | Vanilla JS + Marked.js |
| **Cloudflare Pages** | 静态文件托管、全球 CDN 分发、自动部署 | Cloudflare CDN |
| **Cloudflare Worker** | 点赞数据读写 API、评论数聚合 API | Cloudflare Workers + GitHub REST API |
| **GitHub 主仓库** | 文章源文件存储、版本控制、点赞数据持久化 | Git + GitHub REST API |
| **GitHub 评论仓库** | Giscus 评论数据（Discussions） | GitHub Discussions + Giscus App |

## 二、数据存储方案

### 2.1 文章数据

```
posts/
├── index.json          # 文章元数据索引（站点加载时读取）
├── likes.json          # 点赞数据 { "slug": count }
├── info-and-knowledge.md  # 文章正文（Markdown + Frontmatter）
├── start-blogging.md
├── minimal-design.md
├── frontend-tips.md
├── reading-notes.md
├── about-this-blog.md
└── victim-thinking.md
```

**index.json 结构**：

```json
[
  {
    "slug": "victim-thinking",
    "title": "摆脱受害者思维",
    "date": "2026-06-04",
    "category": "随想",
    "tags": ["灵感"],
    "excerpt": "幸福绝不可能是别人的恩赐"
  }
]
```

**likes.json 结构**（扁平格式）：

```json
{
  "victim-thinking": 1,
  "info-and-knowledge": 2,
  "minimal-design": 1
}
```

### 2.2 评论数据

评论数据完全由 Giscus 管理，存储为 GitHub Discussions。Worker 的 `/api/discussions` 端点仅用于读取评论数，不做写入操作。

## 三、核心流程

### 3.1 读者浏览与点赞流程

```
读者打开 peyblog.com
       │
       ▼
Cloudflare Pages 返回 index.html（CDN 缓存，全球边缘节点）
       │
       ▼
SPA 启动：读取 posts/index.json 获取文章列表
       │
       ▼
并行请求：
  ├── GET /api/likes → Cloudflare Worker → GitHub REST API → likes.json
  └── GET /api/discussions → Cloudflare Worker → GitHub REST API → 评论数
       │
       ▼
渲染首页文章卡片（含点赞数、评论数）
       │
       ▼
读者点击文章 → SPA Hash 路由切换到详情页
       │
       ▼
加载 posts/{slug}.md，Marked.js 渲染为 HTML
       │
       ▼
读者点击 ♡ 点赞：
  1. 前端：localStorage 记录 liked 状态，UI 立即更新
  2. 前端：同步 localStorage 中所有点赞数据
  3. 前端：POST /api/likes { likes: {...} }
  4. Worker：合并数据，通过 GitHub REST API PUT 写入 likes.json
       │
       ▼
读者刷新页面：
  1. SPA 启动，GET /api/likes 获取最新点赞数据
  2. 合并 localStorage 的 liked 状态
  3. 显示正确的点赞数和按钮状态
```

### 3.2 站长写文章流程

```
站长访问 peyblog.com/?edit=1 → 进入管理模式
       │
       ▼
配置 GitHub PAT（首次，localStorage 持久化）
       │
       ▼
点击「✏️ 写文章」→ 进入双栏编辑器
       │
       ▼
填写标题、分类、标签 → 写 Markdown
       │
       ▼
右侧实时预览（Marked.js 渲染）
       │
       ▼
点击「发布文章到 GitHub」：
  1. 前端：构建文章 .md 文件内容（Frontmatter + 正文）
  2. 前端：GitHub Contents API PUT /repos/{repo}/contents/posts/{slug}.md
  3. 前端：读取并更新 posts/index.json（追加新文章）
  4. 前端：GitHub Contents API PUT 更新 posts/index.json
       │
       ▼
GitHub 收到 commit → Cloudflare Pages Webhook → 自动部署（~30s）
       │
       ▼
文章上线，全球 CDN 更新
```

## 四、API 设计概览

### 4.1 Worker API（自定义域名 api.peyblog.com）

| 方法 | 路径 | 描述 | 鉴权 |
|------|------|------|------|
| GET | `/api/likes` | 获取所有文章点赞数 | 无（公开仓库） |
| POST | `/api/likes` | 更新点赞数据（完整覆盖/单次增减） | 服务端 PAT（GITHUB_PAT 环境变量） |
| GET | `/api/discussions` | 获取所有文章评论数 | PAT（读 Discussions） |

### 4.2 GitHub REST API（前端直接调用）

| 方法 | 端点 | 用途 | 鉴权 |
|------|------|------|------|
| GET | `/repos/{repo}/contents/posts/{slug}.md` | 读取文章 | PAT（站长浏览器） |
| PUT | `/repos/{repo}/contents/posts/{slug}.md` | 发布/更新文章 | PAT |
| DELETE | `/repos/{repo}/contents/posts/{slug}.md` | 删除文章 | PAT |
| PUT | `/repos/{repo}/contents/posts/index.json` | 更新文章索引 | PAT |

### 4.3 Giscus 评论

| 配置 | 值 |
|------|-----|
| 仓库 | `YankeeDaddy/My-Blog-Comments-CF` |
| 讨论分类 | Announcements |
| 映射方式 | specific（按页面路径） |
| 主题 | 跟随站点明暗模式 |

## 五、技术选型

| 领域 | 选型 | 理由 |
|------|------|------|
| **托管平台** | Cloudflare Pages | 全球 CDN、免费、自动部署、与 Worker 无缝集成 |
| **运行时** | Cloudflare Workers | 轻量级 API 代理，处理点赞/评论数请求 |
| **前端架构** | 原生 HTML/CSS/JS（SPA） | 零构建依赖、极致轻量、无需 Node.js 开发环境 |
| **Markdown 渲染** | Marked.js | 轻量、性能好、CDN 引入 |
| **评论系统** | Giscus | 基于 GitHub Discussions，免费、无需数据库 |
| **数据持久化** | GitHub REST API（Contents） | Git 版本控制、天然备份、公开透明 |
| **部署触发** | GitHub Webhook → Cloudflare Pages | 推送即部署，无需手动操作 |
| **域名** | Cloudflare DNS | 统一域名管理，Worker 自定义域名 |

## 七、安全模型

```
┌──────────────────────────────────────────────────────────────┐
│                        安全边界                               │
│                                                              │
│  访客浏览器                    │  站长浏览器 (?edit=1)        │
│  ┌────────────────────┐       │  ┌──────────────────────┐   │
│  │ • 读取文章（公开）   │       │  │ • 读取文章（公开）    │   │
│  │ • 点赞（Worker PAT）│       │  │ • 写文章（local PAT）│   │
│  │ • 评论（Giscus OAuth│       │  │ • 删文章（local PAT）│   │
│  │    需要 GitHub 登录）│       │  │ • 配置管理            │   │
│  │ • 无法写入文章      │       │  │                      │   │
│  └────────────────────┘       │  └──────────────────────┘   │
│                               │                              │
│  Cloudflare Worker            │  GitHub API                  │
│  ┌────────────────────┐       │  ┌──────────────────────┐   │
│  │ • GITHUB_PAT（服务端│       │  │ • 公开仓库：读无限制  │   │
│  │   环境变量，不暴露）  │       │  │ • 写入需要 PAT 鉴权  │   │
│  │ • 仅处理点赞/评论数  │       │  │ • Fine-grained PAT   │   │
│  └────────────────────┘       │  │   (Bearer 格式)       │   │
│                               │  └──────────────────────┘   │
└──────────────────────────────────────────────────────────────┘
```

**关键安全原则**：
1. **访客零信任**：无需登录即可浏览和点赞，但点赞写入走服务端 PAT，访客侧无法接触到任何 Token
2. **站长 PAT 仅存本地**：GitHub Token 存储在浏览器 localStorage，仅在站长设备上可用
3. **Worker PAT 隔离**：Cloudflare Worker 的 GITHUB_PAT 作为加密环境变量存储，代码和访客均不可见
4. **最小权限**：
   - 站长 PAT：仅授权 `My-Blog-CF` 仓库的 `Contents` 读写
   - Worker PAT：仅授权 `My-Blog-CF`（Contents）和 `My-Blog-Comments-CF`（Discussions）
