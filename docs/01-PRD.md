## 一、产品概述

### 1.1 产品定位

PeyBlog 是一个**极简主义个人博客系统**，面向希望专注写作内容、不被复杂工具链拖累的个人博主。核心理念是「少即是多」——无需数据库、无需构建工具、无需服务器维护，写 Markdown 即发布。

### 1.2 目标用户

| 用户角色 | 描述 | 核心需求 |
|---------|------|---------|
| **博客站长** | 个人博主，具备基础技术能力 | 写文章、管理内容、自定义样式 |
| **普通读者** | 任意访客，无需登录 | 阅读文章、点赞互动、发表评论 |
| **探索型读者** | 对特定主题感兴趣的访客 | 按分类/标签筛选、搜索文章 |

### 1.3 核心价值主张

- **零运维**：基于 Cloudflare Pages + GitHub，无需服务器
- **写即发**：浏览器内 Markdown 编辑器，一键推送发布
- **极简体验**：单文件架构（~2000 行 HTML），无框架依赖
- **持久互动**：点赞数据跨设备同步，评论基于 GitHub Discussions

## 二、功能需求

### 2.1 读者侧功能

#### F1 — 文章浏览
- **优先级**：P0（核心）
- **描述**：访客可浏览文章列表，查看卡片摘要（标题、日期、分类、标签、摘要）
- **验收标准**：
  - 首页显示文章卡片列表，支持分页
  - 每张卡片显示标题、日期、分类、标签、摘要、点赞数、评论数
  - 点击卡片进入文章详情页

#### F2 — 分类与标签筛选
- **优先级**：P1
- **描述**：访客可按分类（侧边栏）或标签筛选文章
- **验收标准**：
  - 侧边栏显示分类列表及文章计数
  - 点击分类或标签过滤文章列表
  - 显示当前筛选状态，支持清除

#### F3 — 全文搜索
- **优先级**：P2
- **描述**：访客可搜索文章标题和内容
- **验收标准**：
  - 搜索框输入后实时过滤文章列表
  - 支持标题和摘要匹配

#### F4 — 文章点赞
- **优先级**：P0
- **描述**：读者可对文章点赞，点赞数据跨设备持久化
- **验收标准**：
  - 点击 ♡ / ♥ 按钮切换点赞状态
  - 点赞数实时更新（前端增量 + 后台同步）
  - 刷新页面后点赞状态和计数保持
  - 不同设备/浏览器访问同一文章，看到一致的点赞数
  - 游客无需登录即可点赞

#### F5 — 文章评论（Giscus）
- **优先级**：P1
- **描述**：读者可通过 GitHub Discussions 评论文章
- **验收标准**：
  - 文章详情页底部加载 Giscus 评论组件
  - 评论需 GitHub 账号登录
  - 文章卡片显示评论数

#### F6 — 明暗主题
- **优先级**：P2
- **描述**：支持亮色/暗色主题切换
- **验收标准**：
  - 点击主题切换按钮切换明暗模式
  - 主题偏好保存到 localStorage
  - Giscus 评论组件跟随主题切换

### 2.2 站长侧功能

#### F7 — 管理模式入口
- **优先级**：P0
- **描述**：通过 URL 参数 `?edit=1` 进入管理模式
- **验收标准**：
  - 普通访客看不到管理功能
  - 管理模式显示写文章入口、GitHub 配置面板

#### F8 — GitHub 配置
- **优先级**：P0
- **描述**：站长配置 GitHub PAT 和仓库信息，用于文章发布
- **验收标准**：
  - 配置面板输入 Token、仓库地址、分支名
  - 「测试连接」按钮验证配置
  - 配置持久化到浏览器 localStorage

#### F9 — Markdown 编辑器
- **优先级**：P0
- **描述**：双栏 Markdown 编辑器（左侧编辑，右侧实时预览）
- **验收标准**：
  - 支持 Frontmatter 元数据编辑（标题、日期、分类、标签、摘要）
  - 常用 Markdown 快捷插入按钮（标题、加粗、链接、图片、代码块、列表等）
  - 标题修改实时更新 slug
  - 右侧实时渲染 Markdown 预览
  - 「保存草稿到本地」和「发布文章到 GitHub」两个操作
  - 编辑已有文章时预填内容

#### F10 — 文章发布与删除
- **优先级**：P0
- **描述**：通过 GitHub Contents API 推送/删除文章
- **验收标准**：
  - 发布文章：将 `.md` 文件推送到 `posts/` 目录，更新 `posts/index.json`
  - 自动生成 slug（基于标题拼音首字母）
  - 删除文章：从 GitHub 删除 `.md` 文件并更新 `posts/index.json`
  - Cloudflare Pages 自动触发重新部署

#### F11 — TOC 目录导航
- **优先级**：P2
- **描述**：文章详情页自动生成目录
- **验收标准**：
  - 从正文 h2/h3 提取生成浮动 TOC
  - 点击 TOC 项滚动到对应标题
  - 阅读进度高亮当前 TOC 项

## 三、非功能需求

### 3.1 性能
- 首屏加载 < 3 秒（含 Markdown 渲染）
- 文章切换无需整页刷新（SPA 路由）
- API 端点响应时间 < 1 秒

### 3.2 可用性
- 移动端响应式布局
- 支持现代浏览器（Chrome/Firefox/Safari/Edge）
- 明暗主题符合 WCAG AA 对比度标准

### 3.3 安全性
- 管理员 Token 仅存储在浏览器 localStorage，不上传服务器
- 点赞 API 使用服务端 PAT，访客侧不暴露 Token
- 文章推送使用 HTTPS + GitHub API

### 3.4 可维护性
- 单文件架构，无构建依赖，易于理解
- 纯静态托管，零服务器维护
- 文章数据 Git 版本控制，天然备份
