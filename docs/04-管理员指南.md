## 一、快速开始

### 你需要准备的东西

| 项目 | 说明 |
|------|------|
| GitHub 账号 | 拥有 `YankeeDaddy/My-Blog-CF` 和 `YankeeDaddy/My-Blog-Comments-CF` 的访问权限 |
| Cloudflare 账号 | 管理 Pages、Workers、DNS |
| 域名 | `peyblog.com`（已在 Cloudflare DNS 管理） |

### 系统依赖

```
peyblog.com (读者访问)
    ├── Cloudflare Pages → 托管博客静态文件
    ├── Cloudflare Worker → 提供点赞/评论数 API
    │     └── api.peyblog.com
    └── GitHub 仓库
          ├── My-Blog-CF (主仓库：文章 + 点赞数据)
          └── My-Blog-Comments-CF (评论仓库：Giscus Discussions)
```

## 二、首次部署配置

### 2.1 GitHub 仓库创建

如果仓库尚未创建，在 GitHub 上创建以下两个仓库：

| 仓库名 | 用途 | 可见性 |
|--------|------|--------|
| `My-Blog-CF` | 主仓库（博客代码 + 文章） | Public |
| `My-Blog-Comments-CF` | 评论仓库（Giscus） | Public |

### 2.2 Cloudflare Pages 设置

1. 登录 [Cloudflare Dashboard](https://dash.cloudflare.com/)
2. **Workers & Pages** → **Create** → **Pages** → **Connect to Git**
3. 选择 `YankeeDaddy/My-Blog-CF`
4. 构建设置：

| 配置项 | 值 |
|--------|-----|
| **Framework preset** | None |
| **Build command** | *(留空)* |
| **Build output directory** | *(留空)* |
| **Root directory** | *(留空)* |

5. 点击 **Save and Deploy**

### 2.3 Cloudflare Worker 设置

> 点赞系统需要独立 Worker 处理 API 请求。这一配置在初次建立后极少需要改动。

**a) 创建 Worker**

1. Cloudflare Dashboard → **Workers & Pages** → **Create** → **Workers**
2. 名称：`peyblog-likes-api`
3. 模板：Hello World → 然后替换为 `worker.js` 的完整内容
4. 点击 **Deploy**

**b) 设置环境变量**

1. 进入 Worker → **Settings** → **Variables and Secrets**
2. 添加：

| 变量名 | 值 | 类型 |
|--------|-----|------|
| `GITHUB_PAT` | GitHub Fine-grained PAT（见下方） | **Secret**（加密） |

**c) 生成 GitHub PAT**

1. 打开 https://github.com/settings/personal-access-tokens/new
2. 填写：

| 字段 | 值 |
|------|-----|
| Token name | `peyblog-worker` |
| Expiration | **No expiration**（推荐） |
| Repository access | **Only select repositories** |
| 选中仓库 | `YankeeDaddy/My-Blog-CF` |
| | `YankeeDaddy/My-Blog-Comments-CF` |

3. Repository permissions：

| 权限 | 级别 |
|------|------|
| **Contents** | Read and Write |
| **Discussions** | Read-only |

4. 生成并复制 Token

> ⚠️ **重要**：Token 只显示一次！务必立即粘贴到 Worker 环境变量中。

**d) 绑定自定义域名**

1. Worker → **Settings** → **Triggers** → **Custom Domains**
2. 添加：`api.peyblog.com`

### 2.4 Giscus 评论系统配置

如果评论仓库尚未配置 Giscus：

1. 进入 `YankeeDaddy/My-Blog-Comments-CF` 仓库 → **Settings**
2. 开启 **Discussions** 功能
3. 访问 https://github.com/apps/giscus → 安装并授权该仓库
4. 访问 https://giscus.app → 填入 `YankeeDaddy/My-Blog-Comments-CF`
5. 获取 `repoId` 和 `categoryId`
6. 在 `index.html` 中更新 `GISCUS_CONFIG`：

```javascript
const GISCUS_CONFIG = {
    repo: 'YankeeDaddy/My-Blog-Comments-CF',
    repoId: 'R_xxxxxxxxxx',       // ← 替换为实际值
    category: 'Announcements',
    categoryId: 'DIC_xxxxxxxxxx',  // ← 替换为实际值
    // ...
};
```

### 2.5 验证部署

依次验证以下端点：

```bash
# 1. 博客主页
curl -s -o /dev/null -w "%{http_code}" https://peyblog.com

# 2. Worker 点赞 API
curl -s https://api.peyblog.com/api/likes

# 3. Worker 评论数 API
curl -s https://api.peyblog.com/api/discussions
```

预期：三个端点均应返回 HTTP 200。

## 三、进入管理模式

博客的管理功能通过 URL 参数激活，普通访客看不到任何管理界面。

### 操作

在浏览器地址栏访问：

```
https://peyblog.com/?edit=1
```

**进入后的变化**：

| 普通模式 | 管理模式 |
|---------|---------|
| 仅显示导航链接 | 导航栏显示「✏️ 写文章」入口 |
| 无管理面板 | 左侧边栏显示「⚙️ 配置 Token」按钮 |
| 文章卡片无操作按钮 | 每张文章卡片显示「编辑」和「删除」按钮 |

> 💡 管理模式状态不会被 URL 记住。关闭标签页后重新打开 `peyblog.com` 即恢复普通模式。

## 四、配置 GitHub Token

首次使用管理功能前，需要配置 GitHub Personal Access Token。

### 4.1 生成 Token

1. 打开 https://github.com/settings/personal-access-tokens/new
2. Token name：`peyblog-editor`
3. Expiration：建议 **No expiration**
4. Repository access：**Only select repositories** → `YankeeDaddy/My-Blog-CF`
5. Permissions：**Contents → Read and Write**
6. Generate → 复制 Token

### 4.2 在博客中配置

1. 进入管理模式（`?edit=1`）
2. 点击左侧「⚙️ 配置 Token」
3. 填写：

| 字段 | 值 |
|------|-----|
| Personal Access Token | 粘贴上面生成的 Token |
| 仓库地址 | `YankeeDaddy/My-Blog-CF` |
| 分支名 | `main` |

4. 点击「💾 保存配置」
5. 点击「🔗 测试连接」确认连接成功

> ⚠️ Token 仅存储在浏览器 localStorage 中，不会发送到任何第三方服务器。不同设备/浏览器需要分别配置。

## 五、写文章与发布

### 5.1 打开编辑器

管理模式 → 点击导航栏「✏️ 写文章」

### 5.2 编辑器界面

```

│ [日期]        │  日期 · 分类 · 标签     │
│ [分类]        │                      │
│ [标签]        │  渲染后的 Markdown    │

│               │                      │
│ Markdown 正文  │                      │

│ [保存草稿]     │                      │

└───────────────┴──────────────────────┘
```

### 5.3 工具栏快捷操作

| 按钮 | 功能 | 插入内容 |
|------|------|---------|
| **B** | 加粗 | `**文本**` |

### 5.4 填写元数据

| 字段 | 必填 | 说明 |
|------|------|------|
| 标题 | ✅ | 同时用于生成 URL slug |
| 日期 | ✅ | 默认当天，格式 YYYY-MM-DD |
| 分类 | ✅ | 如「随想」「技术」「设计」 |
| 标签 | ❌ | 逗号分隔，如「灵感,写作」 |
| 摘要 | ❌ | 显示在文章卡片上，建议 1-2 句话 |

### 5.5 发布流程

```
点击「🚀 发布文章到 GitHub」
  ↓
1. 构建 Frontmatter + Markdown 正文
2. GitHub Contents API: PUT posts/{slug}.md
3. 读取当前 posts/index.json
4. 追加新文章到 index.json
5. GitHub Contents API: PUT posts/index.json
  ↓
GitHub 收到 commit
  ↓
Cloudflare Pages Webhook 触发
  ↓
自动部署（约 30 秒）
  ↓
✅ 文章上线，全球可访问
```

### 5.6 保存草稿

点击「💾 保存草稿到本地」：
- 草稿存储在浏览器 localStorage
- 不会推送到 GitHub
- 关闭浏览器后可在同一设备恢复
- 适合长文写作暂存

## 六、编辑与删除文章

### 6.1 编辑已有文章

管理模式 → 在文章卡片点击「编辑」按钮 → 进入编辑器，预填现有内容 → 修改后点击「发布文章到 GitHub」

发布操作会自动：
- 覆盖 `posts/{slug}.md`（原地更新）
- 更新 `posts/index.json` 中对应条目

### 6.2 删除文章

管理模式 → 在文章卡片点击「删除」按钮 → 确认对话框 → 文章从 GitHub 删除

删除操作会自动：
- DELETE `posts/{slug}.md`
- 更新 `posts/index.json`（移除对应条目）
- Cloudflare Pages 自动重新部署

> ⚠️ 删除不可逆！GitHub 保留了 git 历史可以恢复，但前端没有撤销功能。

## 七、点赞系统维护

### 7.1 系统架构回顾

```
访客点击 ♡
    ↓
浏览器 localStorage（即时更新）
    ↓
POST /api/likes → Worker → GitHub REST API → likes.json
    ↓
其他访客刷新页面 → GET /api/likes → Worker → 即时返回数据
```

### 7.2 健康检查

定期执行以下命令验证系统正常：

```bash
# 1. 读取当前点赞数据
curl https://api.peyblog.com/api/likes

# 2. 测试写入
curl -X POST https://api.peyblog.com/api/likes \
  -H "Content-Type: application/json" \
  -d '{"articleId":"about-this-blog","action":"like"}'

# 3. 验证即时可读
curl https://api.peyblog.com/api/likes
```

### 7.3 Worker 代码更新

如需更新 Worker 逻辑：

1. 编辑仓库中的 `worker.js`
2. 复制全部内容
3. Cloudflare Dashboard → Workers → `peyblog-likes-api` → 代码编辑器
4. 粘贴替换 → **Deploy**

> ⚠️ Worker 代码不会自动更新！修改 GitHub 仓库中的 `worker.js` 后，必须手动到 Dashboard 中更新。

### 7.4 PAT 过期处理

当 GITHUB_PAT 过期时，POST `/api/likes` 返回 403：

1. 进入 GitHub Token 管理页面
2. 找到 `peyblog-worker` Token
3. 点击 Regenerate → 复制新 Token
4. Cloudflare Dashboard → Workers → `peyblog-likes-api` → Settings → Variables → 编辑 `GITHUB_PAT`
5. 粘贴新 Token → Save → **Deploy**

### 7.5 手动重置点赞数据

编辑仓库中的 `posts/likes.json`（扁平格式）：

```json
{
  "all-posts-count-reset": 0
}
```

GitHub 会自动触发 Pages 重新部署。

## 八、评论系统维护

### 8.1 评论管理入口

所有评论存储为 GitHub Discussions：
- 访问 https://github.com/YankeeDaddy/My-Blog-Comments-CF/discussions
- 可按标题（`#/post/{slug}` 格式）找到对应评论

### 8.2 删除不当评论

1. 在 Discussions 列表中找到对应的讨论
2. 找到要删除的评论
3. 点击评论右上角「...」→ 「Delete」

### 8.3 关闭某篇文章的评论

1. 在 Discussions 中找到对应文章的 Discussion
2. 点击「Close discussion」

> 💡 关闭 Discussion 后 Giscus 仍会显示历史评论，但不再允许新增。

### 8.4 评论数 API 异常排查

如果文章卡片评论数显示为 0：

```bash
# 直接查看 Worker 返回的数据
curl https://api.peyblog.com/api/discussions
```

常见原因：
- **PAT 过期**：和点赞系统共用同一个 GITHUB_PAT → 按 7.4 节更新 Token
- **仓库名不匹配**：Worker 代码中 `repo` 变量是否正确
- **Discussions 未开启**：检查 `My-Blog-Comments-CF` 仓库设置

## 九、自定义域名

### 9.1 域名配置一览

所有域名由 Cloudflare DNS 管理：

| 域名 | 指向 | 用途 |
|------|------|------|
| `peyblog.com` | Cloudflare Pages (`pey-blog`) | 博客主站 |
| `www.peyblog.com` | 重定向到 `peyblog.com` | WWW 跳转 |
| `api.peyblog.com` | Cloudflare Worker (`peyblog-likes-api`) | API 端点 |

### 9.2 添加新域名

如需为博客添加新域名：

1. Cloudflare Dashboard → **Pages** → `pey-blog` → **Custom domains**
2. 添加新域名
3. 确保该域名的 DNS 在 Cloudflare 管理下（或添加 CNAME 记录）

## 十、常见问题排查

### Q1：文章发布后不显示

**症状**：点击「发布文章到 GitHub」成功，但博客上看不到新文章

**排查步骤**：
1. 检查 GitHub 仓库 `posts/` 目录是否有新的 `.md` 文件
2. 检查 `posts/index.json` 是否已更新（包含新文章条目）
3. Cloudflare Dashboard → Pages → `pey-blog` → 查看最近的部署状态
4. 手动触发重新部署：Dashboard → **Retry deployment**

### Q2：点赞数不更新或丢失

**症状**：点赞后刷新页面，点赞数恢复为 0

**排查步骤**：
1. 测试 Worker 端点：`curl https://api.peyblog.com/api/likes`
2. 测试写入：`curl -X POST https://api.peyblog.com/api/likes -H "Content-Type: application/json" -d '{...}'`
3. 检查 PAT 是否过期（POST 返回 403）
4. 检查 Worker 日志：Cloudflare Dashboard → Workers → `peyblog-likes-api` → Logs

**原因参照**：
| 返回码 | 原因 | 解决 |
|--------|------|------|
| 403 | PAT 过期或权限不足 | 更新 PAT（见 7.4 节） |
| 404 | likes.json 不存在 | Worker 会自动创建 |
| 500 | 代码错误 | 检查 Worker 日志 |

### Q3：评论组件不显示

**症状**：文章详情页底部没有评论框

**排查步骤**：
1. 确认 `My-Blog-Comments-CF` 仓库已开启 Discussions
2. 确认 Giscus App 已安装并授权该仓库
3. 确认 `index.html` 中 `GISCUS_CONFIG` 的 `repoId` 和 `categoryId` 正确
4. 浏览器打开 F12 → Console → 搜索 giscus 相关错误

### Q4：管理模式无法进入

**症状**：访问 `?edit=1` 后看不到管理功能

**原因**：`ADMIN_MODE` 依赖 URL 参数 `edit=1`

**解决**：确保 URL 格式为 `https://peyblog.com/?edit=1`（注意问号位置）

### Q5：GitHub Token 测试连接失败

| 错误 | 原因 | 解决 |
|------|------|------|
| Token 无效或已过期 | PAT 已过期 | 重新生成 Token |
| 仓库不存在或无权访问 | 仓库名拼写错误或 PAT 权限不足 | 确认仓库名和 PAT 的 `repo` 权限 |

### Q6：Worker 端点返回 404

**原因**：路由配置或域名绑定问题

**排查**：
1. 确认 `api.peyblog.com` 已绑定到 Worker
2. Cloudflare Dashboard → Workers → `peyblog-likes-api` → Triggers → Custom Domains
3. 确认 Worker 代码中有对应的路由处理
